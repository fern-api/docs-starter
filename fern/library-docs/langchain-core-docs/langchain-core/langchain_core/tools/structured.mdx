<Badge>classmethod</Badge>

Create tool from a given function.

A classmethod that helps to create a tool from a function.

**Parameters:**

<ParamField path="func" type="Callable | None" default="None">
The function from which to create a tool.
</ParamField>

<ParamField path="coroutine" type="Callable[..., Awaitable[Any]] | None" default="None">
The async function from which to create a tool.
</ParamField>

<ParamField path="name" type="str | None" default="None">
The name of the tool.

Defaults to the function name.
</ParamField>

<ParamField path="description" type="str | None" default="None">
The description of the tool.

Defaults to the function docstring.
</ParamField>

<ParamField path="return_direct" type="bool" default="False">
Whether to return the result directly or as a callback.
</ParamField>

<ParamField path="args_schema" type="ArgsSchema | None" default="None">
The schema of the tool's input arguments.
</ParamField>

<ParamField path="infer_schema" type="bool" default="True">
Whether to infer the schema from the function's signature.
</ParamField>

<ParamField path="response_format" type="Literal['content', 'content_and_artifact']" default="'content'">
The tool response format.

If `'content'` then the output of the tool is interpreted as the
contents of a `ToolMessage`. If `'content_and_artifact'` then the output
is expected to be a two-tuple corresponding to the `(content, artifact)`
of a `ToolMessage`.
</ParamField>

<ParamField path="parse_docstring" type="bool" default="False">
If `infer_schema` and `parse_docstring`, will attempt
to parse parameter descriptions from Google Style function docstrings.
</ParamField>

<ParamField path="error_on_invalid_docstring" type="bool" default="False">
if `parse_docstring` is provided, configure
whether to raise `ValueError` on invalid Google Style docstrings.
</ParamField>

<ParamField path="**kwargs" type="Any" default="&#123;&#125;">
Additional arguments to pass to the tool
</ParamField>

**Returns:** `StructuredTool`

The tool.

**Raises:**

- `ValueError`: If the function is not provided.
- `ValueError`: If the function does not have a docstring and description
is not provided.
- `TypeError`: If the `args_schema` is not a `BaseModel` or dict.

**Examples:**

<CodeBlock showLineNumbers={false}>

```python
def add(a: int, b: int) -> int:
    """Add two numbers"""
    return a + b
tool = StructuredTool.from_function(add)
tool.run(1, 2) # 3
```
