Message from an AI.

An `AIMessage` is returned from a chat model as a response to a prompt.

This message represents the output of the model and consists of both
the raw output as returned by the model and standardized fields
(e.g., tool calls, usage metadata) added by the LangChain framework.


<ParamField path="content_blocks" type="list[ContentBlock]">
Return standard, typed `ContentBlock` dicts from the message.

If the message has a known model provider, use the provider-specific translator
first before falling back to best-effort parsing. For details, see the property
on `BaseMessage`.
</ParamField>

<ParamField path="invalid_tool_calls" type="list[InvalidToolCall] = Field(default_factory=list)">

</ParamField>

<ParamField path="lc_attributes" type="dict">

</ParamField>

<ParamField path="tool_calls" type="list[ToolCall] = Field(default_factory=list)">

</ParamField>

<ParamField path="type" type="Literal['ai'] = 'ai'">

</ParamField>

<ParamField path="usage_metadata" type="UsageMetadata | None = None">

Breakdown of input token counts.

Does *not* need to sum to full input token count. Does *not* need to have all keys.

May also hold extra provider-specific keys.

!!! version-added "Added in `langchain-core` 0.3.9"

<ParamField path="audio" type="int">
Audio input tokens.
</ParamField>

<ParamField path="cache_creation" type="int">
Input tokens that were cached and there was a cache miss.

Since there was a cache miss, the cache was created from these tokens.
</ParamField>

<ParamField path="cache_read" type="int">
Input tokens that were cached and there was a cache hit.

Since there was a cache hit, the tokens were read from the cache. More precisely,
the model state given these tokens was read from the cache.
</ParamField>

**Bases:** `typing.TypedDict`

Usage metadata for a message, such as token counts.

This is a standard representation of token usage that is consistent across models.

!!! warning "Behavior changed in `langchain-core` 0.3.9"

    Added `input_token_details` and `output_token_details`.

!!! note "LangSmith SDK"

    The LangSmith SDK also has a `UsageMetadata` class. While the two share fields,
    LangSmith's `UsageMetadata` has additional fields to capture cost information
    used by the LangSmith platform.

<ParamField path="input_token_details" type="NotRequired[InputTokenDetails]">
Breakdown of input token counts.

Does *not* need to sum to full input token count. Does *not* need to have all keys.
</ParamField>

<ParamField path="input_tokens" type="int">
Count of input (or prompt) tokens. Sum of all input token types.
</ParamField>

<ParamField path="output_token_details" type="NotRequired[OutputTokenDetails]">
Breakdown of output token counts.

Does *not* need to sum to full output token count. Does *not* need to have all keys.
</ParamField>

<ParamField path="output_tokens" type="int">
Count of output (or completion) tokens. Sum of all output token types.
</ParamField>

<ParamField path="total_tokens" type="int">
Total token count. Sum of `input_tokens` + `output_tokens`.
</ParamField>

</Indent>
