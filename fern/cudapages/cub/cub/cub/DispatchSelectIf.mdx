Utility class for dispatching the appropriately-tuned kernels for [DeviceSelect](/library/api/cub::_device_select) and [DevicePartition](/library/api/cub::_device_partition).

<AccordionGroup>
<Accordion title="Template parameters">

<ParamField path="InputIteratorT" type="typename">
Random-access input iterator type for reading input items
</ParamField>

<ParamField path="FlagsInputIteratorT" type="typename">
Random-access input iterator type for reading selection flags (NullType* if a selection functor or discontinuity flagging is used for selection)
</ParamField>

<ParamField path="SelectedOutputIteratorT" type="typename">
Random-access output iterator type for writing selected items
</ParamField>

<ParamField path="NumSelectedIteratorT" type="typename">
Output iterator type for recording the number of items selected
</ParamField>

<ParamField path="SelectOpT" type="typename">
Selection operator type (NullType if selection flags or discontinuity flagging is used for selection)
</ParamField>

<ParamField path="EqualityOpT" type="typename">
Equality operator type (NullType if selection functor or selection flags are used for selection)
</ParamField>

<ParamField path="OffsetT" type="typename">
Signed integer type for global offsets
</ParamField>

<ParamField path="SelectionOpt" type="SelectImpl">
[SelectImpl](/library/api/cub::SelectImpl) indicating whether to partition, just selection or selection where the memory for the input and output may alias each other.
</ParamField>

<ParamField path="PolicyHub" type="typename" default="detail::select::policy_hub<    detail::it_value_t<InputIteratorT>,    detail::it_value_t<FlagsInputIteratorT>,        ::cuda::std::conditional_t<SelectionOpt == SelectImpl::Partition, OffsetT, detail::select::per_partition_offset_t>,    detail::select::is_partition_distinct_output_t<SelectedOutputIteratorT>::value,    SelectionOpt>">
</ParamField>

</Accordion>
</AccordionGroup>
