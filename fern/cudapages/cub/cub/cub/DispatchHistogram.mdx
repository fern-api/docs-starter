Utility class for dispatching the appropriately-tuned kernels for [DeviceHistogram](/library/api/cub::_device_histogram).

<AccordionGroup>
<Accordion title="Template parameters">

<ParamField path="NUM_CHANNELS" type="int">
Number of channels interleaved in the input data (may be greater than the number of channels being actively histogrammed)
</ParamField>

<ParamField path="NUM_ACTIVE_CHANNELS" type="int">
Number of channels actively being histogrammed
</ParamField>

<ParamField path="SampleIteratorT" type="typename">
Random-access input iterator type for reading input items (may be a simple pointer type)
</ParamField>

<ParamField path="CounterT" type="typename">
Integer type for counting sample occurrences per histogram bin
</ParamField>

<ParamField path="LevelT" type="typename">
Type for specifying bin level boundaries
</ParamField>

<ParamField path="OffsetT" type="typename">
Signed integer type for global offsets
</ParamField>

<ParamField path="PolicyHub" type="typename" default="void">
Implementation detail, do not specify directly, requirements on the content of this type are subject to breaking change.
</ParamField>

<ParamField path="SampleT" type="typename" default="cub::detail::it_value_t<SampleIteratorT>">
</ParamField>

<ParamField path="KernelSource" type="typename" default="detail::histogram:: DeviceHistogramKernelSource<NUM_CHANNELS, NUM_ACTIVE_CHANNELS, SampleIteratorT, CounterT, LevelT, OffsetT, SampleT>">
</ParamField>

<ParamField path="KernelLauncherFactory" type="typename" default="CUB_DETAIL_DEFAULT_KERNEL_LAUNCHER_FACTORY">
</ParamField>

</Accordion>
</AccordionGroup>

```cpp showLineNumbers={false}
template <typename MaxPolicyT>

    void *d_temp_storage,
    size_t &temp_storage_bytes,
    SampleIteratorT d_samples,
    ::cuda::std::array<CounterT *, NUM_ACTIVE_CHANNELS> d_output_histograms,
    ::cuda::std::array<int, NUM_ACTIVE_CHANNELS> num_output_levels,
    ::cuda::std::array<const LevelT *, NUM_ACTIVE_CHANNELS> d_levels,
    OffsetT num_row_pixels,
    OffsetT num_rows,
    OffsetT row_stride_samples,
    cudaStream_t stream,

    KernelSource kernel_source = {},
    KernelLauncherFactory launcher_factory = {},
    MaxPolicyT max_policy = {}
)
```
</CodeBlock>

**Parameters**

<ParamField path="d_temp_storage" type="void *">
Device-accessible allocation of temporary storage. When nullptr, the required allocation size is written to `temp_storage_bytes` and no work is done.
</ParamField>

<ParamField path="temp_storage_bytes" type="size_t &">
Reference to size in bytes of `d_temp_storage` allocation
</ParamField>

<ParamField path="d_samples" type="SampleIteratorT">
The pointer to the multi-channel input sequence of data samples. The samples from different channels are assumed to be interleaved (e.g., an array of 32-bit pixels where each pixel consists of four RGBA 8-bit samples).
</ParamField>

<ParamField path="d_output_histograms" type="::cuda::std::array<CounterT *, NUM_ACTIVE_CHANNELS>">
The pointers to the histogram counter output arrays, one for each active channel. For channel<sub>i</sub>, the allocation length of `d_histograms[i]` should be `num_output_levels[i] - 1`.
</ParamField>

<ParamField path="num_output_levels" type="::cuda::std::array<int, NUM_ACTIVE_CHANNELS>">
The number of boundaries (levels) for delineating histogram samples in each active channel. Implies that the number of bins for channel<sub>i</sub> is `num_output_levels[i] - 1`.
</ParamField>

<ParamField path="d_levels" type="::cuda::std::array<const LevelT *, NUM_ACTIVE_CHANNELS>">
The pointers to the arrays of boundaries (levels), one for each active channel. Bin ranges are defined by consecutive boundary pairings: lower sample value boundaries are inclusive and upper sample value boundaries are exclusive.
</ParamField>

<ParamField path="num_row_pixels" type="OffsetT">
The number of multi-channel pixels per row in the region of interest
</ParamField>

<ParamField path="num_rows" type="OffsetT">
The number of rows in the region of interest
</ParamField>

<ParamField path="row_stride_samples" type="OffsetT">
The number of samples between starts of consecutive rows in the region of interest
</ParamField>

<ParamField path="stream" type="cudaStream_t">
CUDA stream to launch kernels within. Default is stream<sub>0</sub>.
</ParamField>

</Tab>
